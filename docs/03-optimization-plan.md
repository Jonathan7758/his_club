# 03 — 面向历史领域的定向优化方案

## 1. 数据源适配

### 1.1 MindSpider 历史领域爬虫源替换

BettaFish的MindSpider默认覆盖微博、小红书、抖音等泛社媒。需要增加以下历史垂直源：

| 平台 | 类型 | 抓取策略 | 优先级 |
|------|------|---------|--------|
| **知乎** | 历史话题+回答 | 按话题爬取热门回答+评论 | 高 |
| **B站** | 历史类UP主视频数据 | 按分区+UP主爬取播放/弹幕/评论 | 高 |
| **微博** | 历史超话+热搜关联 | 保留，增加历史领域关键词过滤 | 高 |
| **豆瓣** | 历史书评+历史小组 | 按标签爬取书评和讨论 | 中 |
| **抖音/快手** | 历史类视频飙升榜 | 保留MediaEngine多模态能力 | 中 |
| **微信公众号** | 历史类头部账号 | RSS/搜狗微信搜索 | 中 |
| **百度指数** | 历史关键词搜索趋势 | API获取 | 低 |

### 1.2 学术与专业源接入（InsightEngine）

- **知网/维普** — 学术热点发现（通过公开API或RSS）
- **国家文物局/各省博物馆官网** — 展览/考古动态
- **中国历史研究院** — 权威发布

## 2. 三级时间窗口策略

### 2.1 1周窗口 — 日扫描器

**目标**：捕获爆发性热点

```
每日运行 MindSpider → 抓取最新24h内容 → 情感分析 → 热度排序 → Top 10 推送
```

**关键指标**：搜索量增长率、评论增速、平台分布

### 2.2 1月窗口 — 周汇总器

**目标**：识别持续性讨论和上升趋势

```
每周运行 → 拉取近30天数据 → ForumEngine辩论分析 → 趋势报告
```

**关键指标**：7日/30日增长斜率、用户留存讨论度

### 2.3 1年窗口 — 月前瞻器

**目标**：年度热点日历+趋势预测

```
每月运行 → 拉取近365天数据 → 年度热点日历 → 选题储备池
```

**输出**：未来3个月的"选题日历"（含纪念日、大展、大剧上映等）

## 3. 历史领域7维分析框架

ForumEngine 的 Agent 配置需从通用舆情分析切换为历史分析：

### 3.1 7个专业Agent定义

| Agent | 专注领域 | 工具集 |
|-------|---------|--------|
| **政治Agent** | 制度变迁、权力结构、中央地方关系 | 制度史数据库、政策文献 |
| **经济Agent** | 财政、贸易、生产方式的演变 | 经济史数据、计量模型 |
| **军事Agent** | 战争推演、军事技术、战略战术 | 军事地理GIS、兵力分析 |
| **文化Agent** | 思想传播、文化融合、艺术发展 | 文化人类学模型、传播路径 |
| **人物Agent** | 关键人物的决策心理与行为逻辑 | 传记数据库、决策心理模型 |
| **科技Agent** | 技术演进路径与社会影响 | 技术史时间线、溢出效应模型 |
| **地理Agent** | 地理约束、气候变迁、人口迁移 | 历史地图GIS、气候重建数据 |

### 3.2 辩论流程优化

与传统舆情分析的实时性不同，历史分析更强调**深度**而非速度：

```
1. 主持人提出选题 → 给7个Agent 3轮独立研究（非实时并行）
2. 每轮之间 Agent 通过 forum_reader 阅读他人成果
3. 第4轮起进入辩论，主持人引导碰撞
4. 最终输出：3-5个差异化角度 + 每个角度的论据链
```

## 4. 情感分析的领域微调

### 4.1 历史领域情感分析的特殊性

- 历史讨论的情感表达更复杂（不是简单的正面/负面）
- 需要识别"怀古""遗憾""争议""致敬"等历史特有情感维度

### 4.2 微调方案

在 BettaFish 的 SentimentAnalysisModel 基础上：
- 新增**历史情感维度**（怀古、争议、猎奇、共情、反思）
- 使用知乎/B站历史评论区数据微调
- 优先使用 Qwen3 小参数微调方案

---

## 5. 内容质量保障

### 5.1 事实校验层

历史内容的致命伤是事实错误。需要增加：

- **事实核查Agent**：在ReportEngine生成后、发布前，交叉校验关键事实
- **来源追溯**：每个事实点标注出处
- **争议标注**：学术上存在分歧的观点自动标注"存在争议"

### 5.2 审核机制

- **合规审查**：涉政治敏感内容的自动过滤
- **原创性检测**：防止AI生成与已有内容高度雷同
