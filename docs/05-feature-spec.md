# 05 — 功能方案

## 功能总览

| 模块 | 功能 | 优先级 | 依赖 |
|------|------|--------|------|
| F1 热点扫描 | 1周/1月/1年三级时间窗口 | P0 | MindSpider |
| F2 深度分析 | 历史7维辩论+多角度输出 | P0 | ForumEngine |
| F3 轻量预测 | "历史如果"推演选题 | P1 | MiroFish Lite |
| F4 内容生成 | 公众号图文+视频号脚本 | P0 | ReportEngine |
| F5 数据看板 | 阅读量/完读率监控 | P2 | 管道数据回传 |

---

## F1 — 热点扫描器

### F1.1 日扫描器（1周窗口）

```
触发：每日 6:00 AM（HK时间）
流程：MindSpider爬取 → 情感分析 → 热度排序 → 推送Top10
输出：每日"历史热点快报"（微信/钉钉推送 + Web看板）
```

**热度算法**：
```
Score = 0.4 × 搜索量Z-score + 0.3 × 评论增速 + 0.2 × 跨平台分布 + 0.1 × 情感极化度
```

### F1.2 周汇总器（1月窗口）

```
触发：每周一 8:00 AM
流程：拉取30天数据 → ForumEngine辩论分析 → 趋势识别
输出：本周"历史选题推荐Top 20" + 趋势预测报告
```

### F1.3 月前瞻器（1年窗口）

```
触发：每月1日
流程：拉取365天数据 → 年度热点日历 → 选题储备池
输出：未来3个月"选题日历"（标注纪念日、大展、大剧等）
```

## F2 — 历史7维深度分析器

### 输入方式

- **主动触发**：运营从热点列表中选择选题
- **自动触发**：日扫描器中热度 > 阈值 自动进入深度分析
- **手动输入**：运营自行输入选题

### 分析流程

```
选题输入
  │
  ▼
QueryEngine 全网搜索 ──┐
MediaEngine 多模态分析 ─┼── 并行采集基础信息
InsightEngine 数据库挖掘─┘
  │
  ▼
ForumEngine 7维辩论 (3轮独立研究 + 2轮交叉辩论)
  │
  ▼
角度提取 → 每维输出1-2个差异化角度
  │
  ▼
事实校验层 → 交叉验证关键事实
  │
  ▼
输出：选题角度报告（包含论据链、事实来源、争议标注）
```

### 输出格式

```json
{
  "topic": "安史之乱",
  "angles": [
    {
      "dimension": "经济",
      "angle": "均田制的崩溃：一场被忽视的财政灾难",
      "thesis": "安史之乱的根源不是军事问题而是财政问题...",
      "evidence_chain": ["证据1", "证据2", "证据3"],
      "novelty_score": 0.87,
      "controversy": false
    }
  ],
  "recommended_primary_angle": "经济维度",
  "meta": {
    "debate_rounds": 5,
    "participating_agents": 7
  }
}
```

## F3 — "历史如果"轻量预测

### 触发方式

- 运营手动选择（从深度分析结果中）
- 每月最多 4 次（成本考量）

### 简化版 MiroFish 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| Agent数量 | 50-100 | 代表不同阶层/立场的人物 |
| 模拟轮次 | 20-30 | 每次推演的时间步长 |
| 初始变量 | 1-3个 | "如果XX不/发生" |
| 记忆系统 | Zep Cloud免费版 | 基础记忆管理 |

### 输出

- 推演叙事（3-5种可能走向）
- 关键转折点分析
- 对比现实历史的分岔点
- 公众号选题建议

## F4 — 内容原料生成器

### F4.1 公众号图文模板

```markdown
## {标题：由AI生成3个候选}

### 导语
{150-200字，设置悬念或冲突}

### 一、{小节标题}
{300-500字正文，含引用和数据}

### 二、{小节标题}
...

### 结语
{100字收尾，引出思考}

---
📚 推荐延伸阅读：
- {书名/文章链接1}
- {书名/文章链接2}

💬 你怎么看{话题}？欢迎留言讨论。
```

### F4.2 视频号脚本模板

```
| 时长 | 画面 | 旁白 | 备注 |
|------|------|------|------|
| 0-15s | 历史画面+悬念文字 | "公元755年，盛唐的崩塌..." | 钩子 |
| 15-45s | 地图动画 | "从关中的视角看..." | 核心论点1 |
| 45-75s | 人物画像 | "李隆基的决策..." | 核心论点2 |
| 75-105s| 数据图表 | "均田制的崩溃数据..." | 论据支撑 |
| 105-120s| 文字定格 | "所以，帝国崩塌早有伏笔" | 总结金句 |
```

### F4.3 完读率优化机制

内容生成时注入以下策略：
- **前三行钩子**：前100字必须有悬念/冲突/反常识
- **段落长度限制**：每段≤200字，适配手机阅读
- **互动引导**：在40%和70%位置插入互动问题
- **节奏控制**：深→浅→深→浅的交替节奏

## F5 — 数据看板

### 指标

| 指标 | 来源 | 频率 |
|------|------|------|
| 阅读量 | 公众号后台API | 每日 |
| 完读率 | 公众号后台API | 每日 |
| 分享率 | 公众号后台API | 每日 |
| 选题类型分布 | 系统内 | 每周 |
| AI辅助 vs 人工对比 | 系统内 | 每月 |

### 核心分析维度

- 什么类型的选题完读率最高？
- 什么时间窗口的热点选题阅读量最高？
- 7维中哪个维度的角度内容最受欢迎？
- "历史如果"类选题的分享率是否高于传统选题？
