## 架构偏差分析

### 1. 精简了但验证有效的

| 设计 | 实际 | 影响 |
|------|------|------|
| BettaFish ReportEngine | 直接用DeepSeek生成 | 输出质量相当，更灵活 |
| ForumEngine(7个独立Agent进程) | 单进程内7轮LLM调用 | 简化了部署，效果已验证 |
| Docker部署PostgreSQL | 直接系统安装 | 简化了运维 |
| HK+MY双区 | 火山云单机 | 成本降至0（已有） |

### 2. 未完成但影响大的

| 缺口 | 影响 |
|------|------|
| **时间窗口调度器** | 仍需人工触发，未实现自动化 |
| **豆瓣爬虫** | 缺少书评/文化圈数据源 |
| **DeepSentimentCrawling** | MindSpider浅层扫描已够用，深层爬取未启动 |
| **事实校验层** | LLM生成的历史内容缺真实性保障 |
| **数据看板** | 无法追踪阅读量/完读率反馈 |
| **MiroFish预测** | 缺少"历史如果"差异化选题 |
| **知识图谱** | 无法发现话题密度盲区 |

### 3. 架构正确的决策

- **ForumEngine 7Agent辩论代替单一LLM** → 验证了从20%→100%新颖度
- **搜狗微信搜索代替Bing** → 中文公众号竞品检测精准
- **NewsNow API代替自建爬虫集群** → 零成本获取热点，省去代理IP和反爬对抗
- **单机部署** → 比原双区方案简单且成本归零
