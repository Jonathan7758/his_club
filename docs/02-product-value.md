# 02 — 对历史公众号产品的价值映射

## 1. 核心价值链路

```
热点捕获 ──▶ 多角度深度分析 ──▶ 内容原料生成 ──▶ 公众号/视频号管道
(BettaFish)   (BettaFish)        (BettaFish+MiroFish)   (输出层)
    ▲                                  │
    └────── MiroFish 预测引擎 ─────────┘
         (历史推演/如果选题)
```

## 2. 四大场景价值

### 2.1 热点发现 — 提升选题精准性

**当前痛点**：历史领域热点分散，人工追踪效率低。

**BettaFish适配方案**：

| 时间窗口 | 热点类型 | 数据源 | 频率 |
|---------|---------|--------|------|
| **1周** | 社交媒体爆发性热点 | 微博历史超话、B站历史区、抖音历史类视频飙升榜 | 日扫描 |
| **1月** | 持续性讨论话题 | 知乎历史话题上升榜、豆瓣历史书评、学术期刊热点 | 周汇总 |
| **1年** | 长期趋势/纪念日 | 博物馆大展、历史纪录片、历史纪念年/事件 | 月前瞻 |

**价值**：从被动选题变为主动预判，在热点爆发前准备好内容。

### 2.2 多角度深度分析 — 建立内容壁垒

**当前痛点**：历史选题角度同质化严重（"5个你不知道的..."模式泛滥）。

**BettaFish ForumEngine 适配**：

```
输入：一个历史选题（如"安史之乱"）
      │
      ▼
┌─────────────────────────────────────────┐
│  ForumEngine 辩论                       │
│  主持人：请从以下7维分析"安史之乱"      │
│                                         │
│  👤 政治Agent：中央与藩镇制度的必然冲突   │
│  👤 经济Agent：均田制崩溃与财政危机       │
│  👤 军事Agent：募兵制改革与节度使军事权   │
│  👤 文化Agent：胡汉融合与身份认同         │
│  👤 人物Agent：李隆基个人决策演化轨迹     │
│  👤 科技Agent：信息传递延迟与指挥链断裂   │
│  👤 地理Agent：关中本位与东北边防的矛盾   │
│                                         │
│  ★ 碰撞输出：至少3个非共识角度           │
└─────────────────────────────────────────┘
```

### 2.3 内容原料生成 — 直连工作流管道

**BettaFish ReportEngine 适配**：

- **公众号图文原料**：Markdown/HTML模板，含标题、导语、3-5个小节、金句、推荐延伸阅读
- **视频号脚本原料**：分镜脚本模板（画面描述+旁白文案+时长）

输出格式对接现有管道：
```json
{
  "platform": "wechat_mp",
  "title": "安史之乱：一场本可避免的帝国崩塌",
  "sections": [...],
  "golden_quotes": [...],
  "references": [...],
  "script": {...}  // 视频号脚本
}
```

### 2.4 "历史如果" 预测 — 开拓全新选题类型

**MiroFish 简化版适配**：

50-100 Agent 轻量模拟用于：
- "如果岳飞没被召回..." → 模拟南宋军政走向
- "如果郑和继续远航..." → 模拟全球格局变化
- "如果光绪变法成功..." → 模拟中国近代化路径

这种推演型内容在公众号领域属于蓝海，差异化极强。
